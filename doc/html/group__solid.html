Parallel Material Point Method Simulator
solid

Syntax

solid(solid-ID, region, region-ID, N_ppc1D, material-ID, h, T0)
solid(solid-ID, mesh, meshfile, material-ID, h, T0)
solid(solid-ID, file, filename, material-ID, h, T0)
  • solid-ID: name of the solid to be created.
  • region-ID: name of the region that occupies the solid.
  • N_ppc1D: equivalent number of particles per cell in 1D (along one axis).
  • material-ID: name of the material to be used by the solid.
  • h: cell-size of the background grid.
  • T0: temperature of the solid at the start of the simulation.
  • meshfile: name of the file containing the mesh.
  • filename: name of the file containing the list of particles as well as their positions.

Examples

E = 115
nu = 0.31
rho = 8.94e-06
ppc = 1
R = 0.2
xc = R/2
yc = R/2
zc = 0
region(rBall, sphere, xc, yc, zc, R)
material(mat1, linear, rho, E, nu)
solid(sBall, region, rBall, ppc, mat1, R / 10, 25)

Defines a solid called 'sBall' delimited by region 'rBall', a sphere (Sphere) of radius centered around (xc, yc, zc) which constitutive law (Material) is linear elastic.

Description

A solid is a collection of particles. These particles are created according to either:

  1. a region: the region delimits the extends of the solid, and the particles are created within it,
  2. a mesh: the particles correspond to the integration points of the mesh,
  3. a file: the particles taken from the list found in the file.

When the solid is delimited by a region, the number of particles occupying a given background grid cell is given by the N_ppc1D parameter.
